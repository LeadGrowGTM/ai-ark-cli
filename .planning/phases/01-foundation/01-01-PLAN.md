Create comprehensive TypeScript types derived from real API responses. **Types must match the actual JSON shapes in docs/api-reference/examples/**.

1. **src/types/common.ts** — Shared types:
   - `ApiError`: `{ timestamp: string; status: number; error: string; path: string }`
   - `Pageable`: `{ pageNumber: number; pageSize: number; sort: { empty: boolean; sorted: boolean; unsorted: boolean }; offset: number; paged: boolean; unpaged: boolean }`
   - `PaginatedResponse<T>`: `{ content: T[]; pageable: Pageable; totalPages: number; last: boolean; totalElements: number; size: number; number: number; sort: { empty: boolean; sorted: boolean; unsorted: boolean }; first: boolean; numberOfElements: number; empty: boolean }`
   - `PaginatedPeopleResponse<T>`: extends PaginatedResponse adding `trackId: string`
   - Filter types from API docs:
     - `FilterWithAllAny`: `{ all?: string[]; any?: string[] }`
     - `FilterWithAllAnyPlusSearchMatch`: `FilterWithAllAny & { searchMode?: "SMART" | "WORD" | "STRICT" }`
     - `RangeWithType`: `{ all?: number[][]; any?: number[][] }`
   - `DateRange`: `{ start: string | null; end: string | null }`

2. **src/types/responses.ts** — Response types (MUST match real example JSON shapes exactly):
   - `CreditResponse`: `{ total: number }`
   - `CompanySummary`: full shape from company-search-response.json including name, description, founded_year, type, industry, staff (total + range), logo
   - `CompanyLink`: website, domain, domain_ltd, linkedin
   - `Company`: id, summary (CompanySummary), link (CompanyLink), industries, keywords, languages, last_updated
   - `CompanySearchResponse`: `PaginatedResponse<Company>`
   - `PersonProfile`: first_name, last_name, full_name, headline, title, picture, background, birth_date, summary
   - `PersonLink`: linkedin, twitter, github (nullable), facebook (nullable)
   - `PersonLocation`: default, short, country, state (nullable), city, position (nullable)
   - `PersonLanguages`: primary_locale, supported_locales, profile_languages (nullable)
   - `Education`: school (id, name, logo, url), degree_name, field_of_study, grade, date (DateRange)
   - `Certification`: name, authority, url, license_number (nullable), display_source, company, date (DateRange)
   - `PositionGroup`: company (id, name, logo, url, employees), date (DateRange), profile_positions array
   - `MemberBadges`: premium, creator, open_to_work, hiring, verified, influencer
   - `PersonDepartment`: departments, sub_departments, functions, seniority
   - `Person`: id, identifier, profile, link, location, languages, industry, educations, certifications, position_groups, skills, member_badges, company (Company), department, last_updated
   - `PeopleSearchResponse`: `PaginatedPeopleResponse<Person>`
   - `ReverseLookupResponse`: person record (same Person shape)
   - `MobilePhoneResponse`: phone records
   - `PersonalityAnalysisResponse`: analysis results
   - `ExportStatistics`: `{ state: "IN_PROGRESS" | "DONE" | "FAILED"; total: number; found: number; notFound: number }`
   - `EmailFinderStatistics`: same shape as ExportStatistics

3. **src/types/requests.ts** — Request body types:
   - `AccountFilter`: domain, linkedin, url, name, socialMediaLink, phoneNumber, industries, location, productAndServices, socialMedia, type, foundedYear, language, geoLocation, employeeSize, revenue, funding, keyword, metric, technologies, naics — all optional, using filter types from common.ts
   - `ContactFilter`: profile filters (fullName, socialMediaLink, linkedin, location, languageSkills, profileBadge), professional filters (currentCompany, pastCompany, seniority, department, experience), skills and education filters — all optional
   - `CompanySearchRequest`: `{ page: number; size: number; lookalikeDomains?: string[]; account?: AccountFilter }`
   - `PeopleSearchRequest`: `{ page: number; size: number; account?: AccountFilter; contact?: ContactFilter }`
   - `ReverseLookupRequest`: `{ kind: "CONTACT"; search: string }`
   - `MobilePhoneRequest`: `{ linkedinUrl?: string; domain?: string; name?: string; type: string }`
   - `PersonalityAnalysisRequest`: `{ url: string }` (LinkedIn profile URL)
   - `ExportPeopleRequest`: same filters as PeopleSearchRequest + webhook options
   - `EmailFinderRequest`: `{ trackId: string }`

4. **src/types/api.ts** — Barrel export file:
